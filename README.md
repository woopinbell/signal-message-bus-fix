# signal-message-bus

`signal-message-bus`는 두 프로세스 사이에서 짧은 문자열을 전달하는 POSIX C 프로그램입니다. 서버와 클라이언트는 `SIGUSR1`, `SIGUSR2`를 각각 비트 값 0과 1로 사용합니다.

공용 헤더에는 프로토콜에서 함께 사용할 상수와 함수 선언을 모아 두었습니다.

## 빌드

```sh
make
```

빌드하면 `server`와 `client` 실행 파일이 생성됩니다. 서버는 시작할 때 자신의 PID를 출력하고 수신을 기다립니다. 클라이언트는 서버 PID와 메시지를 인자로 받습니다.

## 사용

먼저 서버를 실행합니다.

```sh
./server
```

서버가 출력한 PID를 확인한 뒤 다른 터미널에서 메시지를 보냅니다.

```sh
./client 12345 "hello"
```

클라이언트는 각 바이트를 상위 비트부터 차례로 시그널에 담아 보냅니다. 서버는 여덟 비트를 모아 원래 바이트를 복원하고 표준 출력에 기록합니다.

## 메시지 종료

문자열 뒤의 NUL 바이트까지 전송해 한 메시지의 끝을 표시합니다. 서버는 NUL을 받으면 줄바꿈을 출력하고 다음 메시지를 받을 준비를 합니다. 빈 문자열도 같은 규칙으로 전달할 수 있습니다.

## 비트 확인 응답

서버는 한 비트를 처리할 때마다 송신자에게 ACK 시그널을 보냅니다. 클라이언트는 ACK를 확인한 뒤 다음 비트를 보내므로, 표준 시그널이 합쳐져 데이터가 빠지는 일을 줄입니다.

## 응답 시간 초과

ACK가 정해진 시간 안에 도착하지 않으면 클라이언트는 전송을 중단하고 오류 메시지를 출력합니다. 존재하지 않는 PID나 응답하지 않는 프로세스를 대상으로 무기한 기다리지 않습니다.

## 전송 세션

서버는 메시지를 보내는 클라이언트의 PID를 세션 소유자로 기록합니다. 한 메시지를 받는 동안 다른 클라이언트가 보낸 비트에는 거부 응답을 보내므로 서로 다른 메시지의 비트가 섞이지 않습니다.

## 검증

```sh
make test
```

실제 서버와 클라이언트를 실행해 일반 문자열, 빈 문자열, 긴 문자열, 연속 메시지와 UTF-8 바이트열을 확인합니다. 인자가 잘못되었거나 서버 PID가 유효하지 않은 경우의 오류 처리도 함께 검사합니다.

## 중단된 전송 복구

메시지를 끝내기 전에 송신자가 종료되면 서버는 해당 PID가 사라졌는지 확인하고 남은 비트와 세션 소유권을 정리합니다. 아직 살아 있는 송신자의 세션은 임의로 빼앗지 않습니다.

클라이언트는 실행 전에 상속받은 시그널 마스크와 관계없이 ACK를 기다릴 수 있도록 대기 마스크를 구성합니다. 예상하지 못한 시그널로 대기가 중단되어도 남은 시간을 계산해 계속 기다립니다.

`tests/session_ownership.sh`는 중단된 세션의 복구, 살아 있는 송신자의 상태 보존, 상속된 시그널 마스크에서의 ACK 수신을 확인합니다.

## 파일 구성

- `include/minitalk.h`: 공용 상수와 함수 선언
- `src/server.c`: 시그널 수신, 세션 관리, 비트 조립
- `src/client.c`: 메시지 전송과 ACK 대기
- `src/parse_pid.c`: 서버 PID 입력 검사
- `src/write_utils.c`: `write` 기반 출력 함수
- `tests/smoke.sh`: 실제 메시지 전송 검사
- `tests/session_ownership.sh`: 세션 수명과 시그널 마스크 검사

## 지원 범위

이 프로그램은 유닉스 시그널을 이용한 단일 서버 프로세스용 전송 예제입니다. TCP나 유닉스 소켓 기반 메시지 큐를 대신하지 않습니다. 여러 송신자가 표준 시그널을 동시에 연속해서 보내는 모든 상황까지 무손실로 보장하지는 않습니다.
